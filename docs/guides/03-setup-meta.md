This guide explains how to configure the Meta Developer app to automatically publish to your Facebook Page and Instagram Business account.

**Estimated time**: 30-45 minutes
**Cost**: Free

## 1. Overview

A single Meta Developer account manages:

- **Facebook Page** → publishing posts with photos
- **Instagram Business** → publishing photos, carousels, and Reels
- **WhatsApp Business** → sending messages to groups (planned)

### Requirements

- A personal Facebook account
- A Facebook Page where you are an administrator
- An Instagram Professional/Business account linked to the Facebook Page

### Verify the Facebook-Instagram Link

Before proceeding, verify that Instagram is linked to your Facebook Page:

1. Go to your **Facebook Page** → **Settings**
2. Look for **"Linked Accounts"** or **"Instagram"**
3. Verify that your Instagram Business account is connected
4. If not, link it here before proceeding

---

## 2. Create the Meta Developer App

1. Go to [developers.facebook.com](https://developers.facebook.com)
2. Log in with the Facebook account that manages your page
3. Click **"My Apps"** (top right) → **"Create App"**
4. Select the type **"Business"** (or "Other" if Business doesn't appear)
5. Fill in:
   - **App name**: `Multi-Poster` (or any name you prefer)
   - **Contact email**: your email address
   - **Business account**: select your Business Manager (if you have one)
6. Click **"Create App"**

---

## 3. Configure Facebook Page Publishing

From the app dashboard:

1. In the **"Use cases for this app"** section, look for:
   - **"Manage everything on your Page"** → click the arrow to expand
   - **"Manage messages and content on Instagram"** → click the arrow
2. If you don't see them, click **"Add use cases"** (top right) and add them

This activates the products needed for publishing.

## 4. Get the Page Access Token

### Via Graph API Explorer

1. Go to the [Graph API Explorer](https://developers.facebook.com/tools/explorer/)
2. In the right panel:
   - **Meta App**: select your app (`Multi-Poster`)
   - **User or Page**: `User Token`
3. In the **"Permissions"** section, add these permissions:
   - `pages_show_list`
   - `pages_manage_posts`
   - `pages_read_engagement`
   - `instagram_basic`
   - `instagram_content_publish`
4. Click **"Generate Access Token"**
5. An authorization window will open:
   - Log in if prompted
   - **IMPORTANT**: When asked to select pages, choose **"Opt in to all current and future Pages"** or check your specific page
   - Continue and complete the authorization
6. Now in the URL field type: `me/accounts` and click **"Submit"**
7. The response will contain an object for each page:

```json
{
  "data": [
    {
      "access_token": "EAAxxxxxxx...",
      "id": "1403696172988698",
      "name": "Your Page Name"
    }
  ]
}
```

8. Copy:
   - **`access_token`**: this is the Page Access Token
   - **`id`**: this is the Facebook Page ID

---

## 5. Find the Instagram Business ID

Using the Page Access Token obtained in the previous step:

1. In the Graph API Explorer, type in the URL bar:
   ```
   {PAGE_ID}?fields=instagram_business_account
   ```
   (replace `{PAGE_ID}` with your page ID)

2. Click **"Submit"**
3. The response will be:

```json
{
  "instagram_business_account": {
    "id": "17841400xxxxxxxx"
  },
  "id": "1403696172988698"
}
```

4. Copy the **`id`** inside `instagram_business_account`: this is your Instagram Account ID

## 6. Long-Lived Token

The token generated by the Graph API Explorer expires after approximately 1-2 hours. To get a long-lived token (60 days):

### Step 1: Extend the User Token

In the Graph API Explorer or your browser, make this request:

```
GET /oauth/access_token?
  grant_type=fb_exchange_token&
  client_id={APP_ID}&
  client_secret={APP_SECRET}&
  fb_exchange_token={SHORT_LIVED_TOKEN}
```

Where:
- `{APP_ID}`: found in app dashboard → Settings → Basic
- `{APP_SECRET}`: found in app dashboard → Settings → Basic
- `{SHORT_LIVED_TOKEN}`: the user token generated earlier

### Step 2: Get the Permanent Page Token

With the extended token, repeat the request:

```
GET /me/accounts?access_token={LONG_LIVED_USER_TOKEN}
```

The page `access_token` in the response will now be a **page token that never expires**.

> **If you use Make.com**: you don't need to do any of this! Make.com manages tokens automatically via OAuth. This procedure is only necessary for the Python script.

## 7. Configuration for Make.com

If you're using Option A (Make.com):

1. In the Facebook Pages module, click **"Create a connection"**
2. Make.com will open the OAuth flow automatically
3. Log in with Facebook and authorize
4. Select your page — done!

For Instagram:

1. In the Instagram for Business module, click **"Create a connection"**
2. Use the same Facebook connection (or create a new one with the same login)
3. Make.com will automatically link the associated IG Professional account

---

## 8. Configuration for Python

If you're using Option B (Python script), add these values to the `.env` file:

```env
META_PAGE_ACCESS_TOKEN=EAAxxxxxxxxxxxxxxxxxxxxxxx
FACEBOOK_PAGE_ID=1403696172988698
INSTAGRAM_ACCOUNT_ID=17841400xxxxxxxx
```

> **Remember**: use the long-lived page token (see [step 6](#6-long-lived-token)) to avoid it expiring after a few hours.

## 9. WhatsApp Cloud API Setup

> **Note**: The WhatsApp Groups API was released in October 2025 and is still relatively new. This section is under active development.

### Basic Setup

1. In your Meta app dashboard, add the **"WhatsApp"** product
2. Follow the guided setup to link your WhatsApp Business number
3. In the WhatsApp section, you will find:
   - **Phone Number ID**: your number's ID
   - **Temporary Access Token**: a temporary token for testing
4. For a permanent token, use a **System User** in Business Manager

### Getting Group IDs

WhatsApp groups managed via Cloud API have a unique JID (format: `xxxxxxxxx@g.us`). To get the IDs:

1. Use the Groups API endpoint: `POST /{phone_number_id}/groups` to list groups
2. Or create/manage groups directly via API

### Recommended Alternatives

If the official Groups API presents issues:

- **[Whapi.cloud](https://whapi.cloud)**: bridge service with a free plan, excellent Make.com integration
- **[Ultramsg](https://ultramsg.com)**: affordable alternative for group messaging

## 10. Troubleshooting

### "me/accounts" returns an empty array

- Verify that you selected the page during authorization
- Re-click "Generate Access Token" and select **"Opt in to all current and future Pages"**
- Verify that the Facebook account you're logged in with is a page administrator

### "OAuthException" - Token expired

- Tokens from the Graph API Explorer expire after 1-2 hours
- Generate a long-lived token (see [step 6](#6-long-lived-token))
- If using Make.com, reauthorize the connection

### Instagram: "The image is not available"

- The image URL must be publicly accessible (not a local file)
- Use Shopify URLs (`cdn.shopify.com/...`) or an image hosting service
- The format must be JPEG or PNG

### Instagram: "Aspect ratio is not supported"

- The aspect ratio must be between **4:5** (portrait) and **1.91:1** (landscape)
- Width must be between 320px and 1440px
- Resize images before uploading them to Shopify

### "instagram_business_account" not present

- The Instagram account is not linked to the Facebook Page
- Go to Facebook → Page → Settings → Instagram → Link Account
- The Instagram account must be of type **Professional** or **Business** (not personal)
